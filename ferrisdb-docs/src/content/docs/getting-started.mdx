## Quick Start

<Steps>

1. **Clone the repository**

   ```bash
   git clone https://github.com/ferrisdb/ferrisdb.git
   cd ferrisdb
   ```

2. **Install Rust toolchain**

   ```bash
   rustup toolchain install stable
   rustup component add rustfmt clippy
   ```

3. **Build the project**

   ```bash
   cargo build --all
   ```

4. **Run tests**
   ```bash
   cargo test --all
   ```

</Steps>

<Aside type="tip">
  The first build will take a few minutes as Cargo downloads and compiles dependencies. Subsequent
  builds will be much faster!
</Aside>

## Exploring the Code

### Start with Storage Engine

The storage engine is the heart of FerrisDB. Here's how to explore it:

1. **Read the WAL implementation**: `ferrisdb-storage/src/wal/`
   - Start with `mod.rs` for the overview
   - Look at `writer.rs` for durability guarantees
2. **Understand the MemTable**: `ferrisdb-storage/src/memtable/`

   - `skip_list.rs` - Lock-free concurrent data structure
   - See how writes are buffered in memory

3. **Explore SSTable format**: `ferrisdb-storage/src/sstable/`
   - `writer.rs` - How data is persisted to disk
   - `reader.rs` - Efficient lookups in sorted files

### Running Examples

```bash
# Create a sample SSTable
cargo run --example create_sstable

# Test WAL recovery
cargo run --example wal_recovery

# Benchmark skip list performance
cargo bench --bench skiplist_bench
```

## Debugging Tips

### Understanding Test Failures

```bash
# Run single test with backtrace
RUST_BACKTRACE=full cargo test test_name -- --nocapture

# Use println! debugging
println!("Value at this point: {:?}", value);

# Use dbg! macro (removes easily)
dbg!(&my_variable);
```

### Performance Investigation

```bash
# Run benchmarks
cargo bench

# Generate flamegraph (requires perf on Linux)
cargo flamegraph --bench bench_name

# Check binary size
cargo bloat --release
```
